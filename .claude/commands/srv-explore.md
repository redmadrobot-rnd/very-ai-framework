Запусти readonly-исследование сервера по вопросу: **$ARGUMENTS**

Следуй скиллу `srv-explore`. Порядок:

1. Убедись, что у сессии есть доступ к нужному окружению (SSH/креды). Для БД нужна
   readonly-роль: если её нет — субагент выдаст готовый рецепт и имя Environment
   Secret, останови и попроси инженера провижинить (сам роль не создавай).
   На prod доступ — через GitHub Environment с подтверждением человека; на dev
   допустим более простой доступ.
2. Спавни субагента `srv-explore` (Task, subagent_type `srv-explore`) с этим
   вопросом и проектной спецификой из `profiles/<project>.md`, если она есть.
3. Верни инженеру findings-отчёт субагента (вопрос → что проверил → находки →
   гипотеза → предложенное действие).

Субагент readonly по построению (tools без Write/Edit + PreToolUse-гард + read-only
роль БД) и ничего не меняет. Если находка требует изменения — субагент вернёт точное
предложенное действие. Выполнить его можешь ты (основной агент) — но **только после
явного одобрения человеком** и доступом инженера, живьём в сессии, не через PR. Одно
одобренное человеком действие → стоп. Без одобрения ничего на сервере не меняй.
