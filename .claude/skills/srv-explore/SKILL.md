# srv-explore — подключение из основного проекта

srv-explore состоит из двух частей:

- **Серверный бандл** (`srv_explore/` в этом репо) — изолированный сервис: remote MCP
  + readonly-агент (Claude Agent SDK) + `guard.py`, режущий всё не-read. Ставится на
  сервер отдельно. Установка/эксплуатация — `srv_explore/README.md`.
- **Этот коннектор** (`.claude/`) — как из своего Claude Code дотянуться до
  запущенного сервиса. Ничего секретного/серверного тут не лежит.

Инженеру **не нужен SSH к серверу** — только bearer-токен, который выдаёт админ.

## 1. Подключиться к серверу (один раз)

Серверный бандл уже развёрнут (см. `srv_explore/README.md`), админ выдал тебе токен.
Добавь remote MCP в свой Claude Code:

```bash
claude mcp add --transport http srv-explore https://<host>/mcp \
  --header "Authorization: Bearer srvx_..."
```

`<host>` — адрес сервиса (обычно за reverse-proxy окружения; напрямую сервис слушает
localhost). Токен привязан к окружению (dev/prod) — dev-токен в prod не пустят.

После этого доступны MCP-инструменты `srv_explore(task)` и `srv_explore_status(job_id)`.

## 2. Пользоваться

- Команда **`/srv-explore <вопрос>`** — обёртка: зовёт инструмент `srv_explore` с
  вопросом, поллит `srv_explore_status` до готовности, отдаёт findings-отчёт.
- Или напрямую: вызвать инструмент `srv_explore` с задачей на естественном языке.

Отчёт: **Вопрос → Что проверил → Находки → Гипотеза → Предложенное действие**.

## 3. Что гарантировано (кратко)

Агент **физически только читает**: readonly-роль БД (фундамент) + `guard.py` на
сервере как граница (не на твоей машине — не обойти) + `permission_mode=dontAsk` +
egress закрыт. Если находка требует изменения — агент его **предлагает**, а выполняет
основной агент твоей сессии по твоему одобрению (живьём, не сам разведчик).
