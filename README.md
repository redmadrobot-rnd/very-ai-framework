# very-ai-framework

Фреймворк для **агентной разработки поверх репозитория**: процесс работы с Codex или
Claude Code, формат технической документации и CI/CD — собраны так, чтобы агент вёл
фичу от идеи до прода вместе с человеком. Три части вложены по смыслу: **dev-flow** —
основа процесса, **KB** и **CI/CD** — возможности, которые он использует.


> Это **референс-репозиторий**: сам фреймворк, без прикладного приложения.
> Содержит инфраструктуру, чтобы агент развернул пайплайн в другом репозитории и вёл его разработку.
>
> Методология фреймворка — в [Miro](https://miro.com/app/board/uXjVHHc75W4=/).

## Засетапить фреймворк агентом

В **целевом** репозитории запусти Codex или Claude Code и поставь задачу — агент сам
изучит проект, перенесёт файлы и настроит:

- **Codex-first:** скопируй `skills/setup-framework-codex/` в `~/.codex/skills/` и
  используй скилл `setup-framework-codex`; либо дай Codex текстовую задачу:

  > Разверни very-ai-framework для Codex по шаблону
  > `github.com/redmadrobot-rnd/very-ai-framework`
  > (инструкция — `skills/setup-framework-codex/SKILL.md`). Целевой репозиторий — текущий.

- **Claude Code:** если скилл `/setup-framework` доступен (поставь репо как плагин или
  скопируй `skills/setup-framework/` в `~/.claude/skills/`) — вызови его; либо дай
  текстовую задачу по `skills/setup-framework/SKILL.md`.

Дальше агент по «Шагу 0» сам спросит недостающее: стенды, `SSH_HOST`/`SSH_USER`, доступы,
нужен ли Codex-ревью (runner/API) и нужен ли `@claude`. Подготовь заранее (или попроси
агента): сервер с Docker и SSH-доступом, GitHub-токен с правами на **secrets** и
**environments**. Подробности пайплайна — в [CICD.md](CICD.md).

> CI/CD здесь — базовый, минимально рабочий каркас. Модифицировать под нужды команды и проекта.

## Структура репозитория

| Путь | Часть | Назначение |
|---|---|---|
| `.github/workflows/`, `.github/scripts/` | CI/CD | воркфлоу и скрипты пайплайна |
| `skills/setup-framework-codex/` | Setup | Codex-инсталлер: разворачивает **весь** фреймворк в другом репо с `AGENTS.md` и `.codex/` |
| `skills/setup-framework/` | Setup | Claude-инсталлер: разворачивает **весь** фреймворк в другом репо. В корне (**не** в `.claude/`) — не догфуд, в проект не копируется |
| `.claude/skills/dev-flow/`, `.claude/skills/task-breakdown/` | Dev-flow | скиллы процесса разработки |
| `.claude/skills/kb-search/`, `.claude/skills/kb-maintain/` | KB | поиск и ведение базы знаний (GitMark) |
| `.claude/commands/` | KB | команды `/kb-doc`, `/kb-build`, `/kb-graph` |
| `docs/gitmark/` | KB | база знаний прикладного проекта (`ontology.md` — спека модели) |
| `AGENTS.md` | Dev-flow | правила разработки для Codex |
| `CLAUDE.md` | Dev-flow | правила разработки для кодового агента |
| `pyproject.toml` | CI/CD | конфиг ruff + pytest |
