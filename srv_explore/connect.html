srv-explore · readonly-разведка сервера

Агент на сервере отвечает на вопросы о его состоянии — только чтением. Подключение занимает пару минут, shell-доступ к серверу не нужен.

1

Сгенерируй SSH-ключ

ssh-keygen -t ed25519 -f ~/.ssh/srvx -N ""
2

Отдай админу публичную часть

Файл ~/.ssh/srvx.pub + как тебя подписать. Админ добавит ключ — токен дальше получишь сам (шаг 4).

3

Подними туннель

Держи открытым, пока работаешь (ключ умеет только этот проброс):

ssh -N -L 8765:localhost:8765 srvx-tunnel@{{HOST}} -i ~/.ssh/srvx
4

Получи токен по своему ключу

Через туннель, доказав владение приватным ключом (challenge → подпись):

KEY=~/.ssh/srvx
N=$(curl -s http://localhost:8765/token/challenge | sed 's/.*"nonce":"//;s/".*//')
printf %s "$N" > /tmp/srvx.n && ssh-keygen -Y sign -f "$KEY" -n srv-explore /tmp/srvx.n
SIG=$(base64 < /tmp/srvx.n.sig | tr -d '\n')
curl -s -X POST http://localhost:8765/token/redeem -H 'Content-Type: application/json' \
  -d "{\"pubkey\":\"$(cat $KEY.pub)\",\"nonce\":\"$N\",\"signature_b64\":\"$SIG\"}"

В ответе — твой srvx_… токен.

5

Подключи MCP в Claude Code

claude mcp add --transport http srv-explore http://localhost:8765/mcp \
  --header "Authorization: Bearer srvx_…"

Дальше — инструмент srv_explore(task): вопрос на естественном языке, в ответ findings-отчёт. Или спрашивай прямо в /admin.

Агент физически не может писать: read-only ФС на уровне ядра, гард на каждую команду, read-only роли БД. Все сессии видны админу. Админка — /admin.
