# Правила overview-сайта

Это **ТЗ разработчика** для агента, который собирает overview-сайт на релизе.
Файл — единственный человеческий вход; всё остальное (onepager, hld, changelog, диаграммы)
агент генерирует сам и кладёт на ветку `overview`. Правь этот файл, чтобы влиять на сайт.

> ⚠️ GitHub Pages публичен по URL (если у репо нет GitHub Enterprise с Pages access control).
> **Не описывай здесь секреты/чувствительное** — оно попадёт на публичный сайт.

## Что обязательно показать
<!-- Перечисли разделы/факты, которые должны быть на сайте всегда. Пример: -->
- Назначение проекта в 2–3 предложениях (для первичного онбординга).
- Текущий статус (MVP / demo / prod / …).
- Технологический стек по группам (frontend / backend / storage / infra).
- High-Level Design: контекст системы и основные компоненты (PlantUML).
- Changelog с подсветкой последнего релиза.
