# AGENTS.md — правила разработки

Что обязан соблюдать агент, работая в этом репозитории.

## Ветки и коммиты
- Ветка от `main`, имя `feature/<суть>`. В `main` напрямую не коммитим.
- Перед каждым коммитом — **pre-commit** (ruff формат+линт, тесты, скан секретов).
  Включить один раз: `pip install pre-commit && pre-commit install`
  (или гонять `pre-commit run` вручную). Любой шаг упал → **не коммитим**, чинимся.
- Не выходим за рамки задачи: объём изменений не должен заметно превышать запланированное.

## Сервисы
- Сервис = каталог `services/<имя>/` со своим `Dockerfile`. Новый сервис — просто новый
  каталог, пайплайн подхватит сам (имена нигде не зашиты).

## Тесты
- Быстрые — в `tests/unit`. Медленные/дорогие (e2e, load) помечать `@pytest.mark.heavy`.
- На push в feature гоняются только лёгкие (`-m "not heavy"`), на PR — все.

## Инструменты и конфиг
- `ruff` запинен (`pyproject.toml` + `.pre-commit-config.yaml` + CI — одна версия).
  Меняешь версию — синхронно во всех трёх местах.
- Правила ruff и длину строки бери из `pyproject.toml`, локально не переопределяй.

## Секреты
- Никогда не коммить ключи/токены/секретные `.env`.
- Несекретный конфиг → Environment **Variables**; секреты → Environment **Secrets**.
